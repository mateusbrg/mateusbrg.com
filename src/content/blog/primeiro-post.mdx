## Por que escolhi Astro para o meu blog?

Depois de anos experimentando diversos frameworks e ferramentas para construir sites, finalmente encontrei no **Astro** a combinação perfeita entre performance, experiência de desenvolvimento e flexibilidade. A proposta do Astro é simples e poderosa: enviar o mínimo de JavaScript possível para o navegador.

Diferente de frameworks como Next.js ou Nuxt, o Astro foi projetado desde o início para sites focados em conteúdo. Isso significa que páginas de blog, documentação e portfólios ganham performance excepcional sem esforço adicional.

## A arquitetura de ilhas

Um dos conceitos mais interessantes do Astro é a **arquitetura de ilhas** (_Islands Architecture_). A ideia é que a maior parte da página é HTML estático, e apenas os componentes que precisam de interatividade são "hidratados" com JavaScript.

> "A melhor forma de otimizar JavaScript é não enviar JavaScript." — Essa é a filosofia central do Astro, e ela faz toda a diferença na experiência do usuário final.

Isso permite misturar componentes de diferentes frameworks na mesma página:

- Componentes **Vue** para interfaces interativas
- Componentes **React** para bibliotecas específicas
- Componentes **Svelte** para animações leves
- E tudo isso coexistindo harmoniosamente

## Configurando o projeto

A configuração inicial do projeto é surpreendentemente simples. Veja como o arquivo de configuração principal fica:

```typescript
import { defineConfig } from "astro/config";
import vue from "@astrojs/vue";
import mdx from "@astrojs/mdx";
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
  integrations: [vue(), mdx()],
  vite: {
    plugins: [tailwindcss()],
  },
});
```

Com poucas linhas, temos suporte a Vue, MDX e TailwindCSS v4. O ecossistema de integrações do Astro facilita muito a adição de funcionalidades sem configuração complexa.

## Content Collections: organizando o conteúdo

O Astro 5 trouxe melhorias significativas para as **Content Collections**. Agora, definimos nossas coleções no arquivo `src/content.config.ts` com tipagem completa e validação via Zod:

```typescript
import { defineCollection } from "astro:content";
import { glob } from "astro/loaders";
import { z } from "astro/zod";

const blog = defineCollection({
  loader: glob({ pattern: "**/*.{md,mdx}", base: "./src/content/blog" }),
  schema: ({ image }) =>
    z.object({
      title: z.string(),
      description: z.string(),
      date: z.date(),
      image: image(),
      tags: z.array(z.string()).optional(),
      draft: z.boolean().default(false),
    }),
});

export const collections = { blog };
```

Essa abordagem garante que todo post tenha os campos obrigatórios preenchidos corretamente, evitando erros em tempo de execução.

## TailwindCSS v4: uma nova era

O TailwindCSS v4 representa uma evolução significativa. Entre as principais mudanças estão:

- **Engine baseado em Rust**: compilação até 10x mais rápida
- **CSS-first configuration**: configuração via `@theme` diretamente no CSS
- **Detecção automática de conteúdo**: sem necessidade de configurar `content` manualmente
- **Novo sistema de cores**: com suporte nativo a P3 wide gamut

A integração com o Astro é feita através do plugin `@tailwindcss/vite`, tornando tudo extremamente fluido.
